Meteor

In this page, we will walk through the basic a simple application that prints “Welcome to CS6320” using Meteor.

Setting up the environment

To begin Meteor application, Install Meteor by following the instruction given here Install

1. Create a Meteor Application

Creating a simple Application to print “Welcome to CS6320” using Meteor. Navigate to a home directory in the terminal and run the following command. first_app is the name of the folder.

  • $ meteor create first_app

Now, change the directory to first_app

  • $ cd first_app

Below is the folder structure that will be created for first_app.

/client - stores all code that runs on client(localhost)

/server - where the logic which runs on the server should go

/public - put all static files over here

client/main.js - loaded on the client by JavaScript entry point
client/main.html - To defines view templates in HTML file
client/main.css - To define app style in CSS
server/main.js - loaded on the server by JavaScript entry point
package.json - Installing NPM packages
meteor - Internal Meteor files
.gitignore - control file for git

2. Review the generated files that is created

2.1 main.js

Once you open main.js file in text editor, you can find JavaScript file that has the script for both client and server( “if (Meteor.isClient)” and “if (Meteor.isServer)” ). As meteor is a full-stack framework, the code that you write is run on both the server and the client. “Session.setDefault” call initializes a section variable called “counter” in the browser, which acts like a global in meteor. “Template.hello.helpers” has the “helper” called “counter” which grabs the session variable’s value and returns it. The session variable is not showing directly in “{{counter}}”. The helper is “reactive”. Meteor and Blaze automatically reruns when session variable gets changed. Meteor re-runs this helper function and Blaze updates the browser with the new content. In this example “click button” is used to identify the event by event type and tells Meteor what that event should do.

  • first_app/client/main.js 

        
      
import { Template } from 'meteor/templating';
import { ReactiveVar } from 'meteor/reactive-var';

import './main.html';

if (Meteor.isClient) {
  // counter starts at 0
  Session.setDefault('counter', 0);

Template.hello.onCreated(function helloOnCreated() {
  // counter starts at 0
  this.counter = new ReactiveVar(0);
});

Template.hello.helpers({
  counter() {
    return Template.instance().counter.get();
  },
});

Template.hello.events({
  'click button'(event, instance) {
    // increment the counter when button is clicked
    instance.counter.set(instance.counter.get() + 1);
  },
});

}
 
if (Meteor.isServer) {
  Meteor.startup(function () {
    // code to run on server at startup
  });
}

2.2 main.html

The HTML file contains the templates and views for the app, both styled by main.css. “Blaze” is a templating engine that Meteor uses to render response from HTML pages. Inside each pair of double braces Blaze examines the expressions and replaces each one with the value.In this example Blaze includes a template called “hello” by “{{> hello}}”. In the <template name="hello"> section, corresponding template is defined.

  • first_app/client/main.html 

        
<head>
  <title>hello </title>
</head>

<body>
  <h1>Welcome to CS6320-Webbased!</h1>
  {{> hello}}
</body>

<template name="hello">
  <button>Click Me</button>
  <p>You've pressed the button {{counter}} times.</p>
</template>

3. Run Meteor Application

3.1 Deploy locally

  • Start the server by give the below command in the project folder

    $ meteor

    Once the server starts running, You should see the following in your console

    [[[[[ ~/Meteor_projects/first_app ]]]]]
    => Started proxy.
    => Started MongoDB.
    => Started your app.
    => App running at: http://localhost:3000/

    open any web browser to view the application running on http://localhost:3000/


3.2 Delpoy on Meteor’s own servers - This application can be even deployed in Meteor’s own servers using the built-in meteor deploy command

  • $ meteor deploy first_app.meteor.com
