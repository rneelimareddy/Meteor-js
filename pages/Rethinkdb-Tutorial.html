Meteor - RethinkDB Tutorial

RethinkDB is a document oriented distributed scalable database which pushes JSON data to the real-time applications. Meteor follows two different types of implementations namely change monitoring + poll-and-diff and oplog tailing, which targets for reactive applications and websites. RethinkDB provides scalable database that makes building realtime applications easy, which Meteor is exactly looking for. One of the special features of RethinkDB is changefeed which is almost similar to Meteor's oplog tailing. For collections in the Meteor-MongoDB architecture sharding is done on database level, it works in the same way for RethinkDB. The only difference between them is that, RethinkDB does not depend on external technical artifacts like oplog, where Meteor does.

Here are some features, which could be implemented when the application is built with Meteor and RethinkDB.

  • On the server, raw access is given to RethinkDB without any ORM's.
  • All the client-side data can be accessed using RethinkDB query language.
  • Meteor's publications/subscriptions model can be used for developing efficient applications.
  • Advantages of Meteor can be taken completely. For example, updating client-side without waiting for the response from server.

    • In this tutorial we are going to learn how Meteor applications can be built and deployed by accessing its data stored in Oracle database. Following are the process steps of how it can be done.

    Installation Steps

    Step-1: Install Oracle database in your local system or get access to external database. You candownload oraclehere.

    Step-2: Install Oracle instant client fromhere. After installing the instant client, run SQL*PLUS by the following commands to find the version of the database.

    • $ sqlplus

      SQL*Plus: Release 12.2.0 Production on Thu Nov 14 15:27:34 2017

      Copyright (c) 1982, 2011, Oracle. All rights reserved.

      Enter user-name: Meteor
      Enter password:

      Connected to: Oracle Database 11g Express Edition Release 12.2.0 - 64bit Production
      SQL>

    Step-3: In the Oracle database, create account for Meteor. It can be done by the following commands.

    • $ sqlplus sys as sysdba
      create user meteor identified by meteor;
      grant connect, resource to meteor;

    Step-4: Check if the Meteor account has been created in the Oracle database. It can be done by the following commands.

    • $ sqlplus Meteor/Meteor

      SQL*Plus: Release 12.2.0 Production on Thu Nov 14 17:46:26 2017

      Copyright (c) 1982, 2011, Oracle. All rights reserved.

      Connected to:Oracle Database 11g Express Edition Release 12.2.0 - 64bit Production
      SQL>exit

    Step-5: Export Meteor-Oracle package to your Meteor project by using the following commands.

    • $ cd PATH-TO-YOUR-METEOR-PROJECT
      $ export LD_LIBRARY_PATH=$LD_LIBRARY_PATH:/instantclient_11_2/
      $ meteor add metstrike:meteor-oracle
      $ meteor

    Example Queries

    1. SQL statement to create table.

    • create table "todos" ("_id" varchar2(17) not null)

    2. SQL statement to alter the table.

    • alter table "todos" add constraint "todos_PK" primary key ("_id")

    3. SQL statement to insert tuples into the table.

    • INSERT: todos { name: 'Get milk', priority: 1, userId: 'Jane', _id: '46662dSw2KkzpNXqo' } { safe: true }

    4. SQL statement to select elements based on condition.

    • SELECT * FROM (SELECT * FROM "todos") WHERE "priority" = 1
