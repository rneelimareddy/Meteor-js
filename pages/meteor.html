About Meteor Js

Meteor was first introduced under the name Skybreak in December 2011 and it was developed by the Meteor Development Group. Later on, Meteor acquired Y Combinator in October 2014 and Kadira in March 2017.Meteor is written using Node.js, It is an open-source and free JavaScript web framework . It is a cohesive development platform with a collection of libraries and packages that are bound together to build extraordinarily dynamic web pages and mobile applications. “Meteor Development Group” has built some libraries like Tracker and Blaze that is for a reactive front-end experience. Meteor can be used with Blaze templating engine, the Angular framework or React library in the client side. Meteor is built using JavaScript file and it is deployed to both the server and the client. Meteor uses the Distributed Data Protocol and a publish–subscribe pattern and it is integrated with Mongo DB.


Components

  • Node.js - JavaScript based web server
  • MongoDB - NoSQL database that "talks" JSON (javascript)
  • Tracker - transparent reactive programming
  • DDP - Distributed Data Protocol "REST for websockets"
  • Blaze/React/Angular - Page templating technologies
  • ISOBuild - a build system for application deployment
  • Other community libraries - standard definition for adding 3rd party packages to Meteor applications

    • key features

  • Communication Layer/The Server – How server interact in Meteor Js

    • Extension to Json is used for serializing and deserialize data moving across the wire via DDP. To reflect changes only on server If (Meteor.isServer) {} is used.

  • Browser/Frontend – Features front end users to interact in Meteor Js

    • To reflecting changes on frontend/browser if (Meteor.isClient) {} is used.

  • Methods – Meteor helps in creating a complex function easier

    • The method is defined in common code on client and server to enable optimistic UI. When the meteor method is defined as a function it is called Basic app. In complex app, few extra features are defined to make it powerful and easily testable. The method is an API endpoint for the server. It can be defined on the server and client, then to call it with some data that is written to the database and get the return value in call-back.

  • Publish – Fetching data from server

    • There can be lot of data which can’t be available to users so for that we tell meteor that publish unflagged posts only and there is no way that client can see flagged posts. This makes meteor app more secure.

  • Subscribe – Using data on client side

    • All the data cannot be send even though the client doesn’t want to see flagged posts. There should be an access to particular data which is subscribed. Any data is subscribed will be mirrored on client (Minimongo ).

  • Helpers – Interaction with the UI

    • To deploy it on UI, helper method is used to provide a display. It is a part of ReactiveContext that is available on each $scope and context. When $scope is destroyed it will stop automatically.

  • Autorun – Checks data modification

    • Autorun establishes a reactive context, the data gets re-executed. So autorun should be limited to a task. For example, there is a Autorun that transforms data when a button is clicked and another retrieves a new page when URL is updated. Both should have separate Autorun, otherwise both will run every time when either scenario is triggered.


    Advantages

  • Developed in one language – Meteor is using JavaScript in all environments It is using pure JavaScript which makes it easy to switch between front- and back-end tasks (application server, web browser, and mobile device)
  • data on the wire - Unlink other web framework, the server sends data (not HTML) and the client renders it.
  • full stack reactivity - without a browser refresh every change made to HTML, CSS, JavaScript on the server are reflected automatically. This is called Live html. As the application’s layers from template to database gets update automatically, it is not necessary to refresh the page to see implemented updates.
  • Latency compensation– When the data is modified on the client, it appears without any delay. Unlike normal server (waiting for a round- trip to the server), Meteor can access data instantaneously. Meteor is extremely important for creating a responsive web application
  • Built-in front-end solution-It Is not necessary to use frameworks such as Angular, Ember. But if there is need, Meteor allows to add another framework like React or Angular to it.
  • Rapid prototyping–Meteor produces cross-platform code (iOS and Android App). Cordova is used to turn web application into smartphone app.
